# 最大化利用技术书籍的建议

下面是我此前分享的一些笔记，针对读者询问如何最大限度地从《从零开始构建大型语言模型》这本书（或系列书籍）中获益。


我自己阅读技术书籍时也遵循类似方法。这并非普适公式，但或许能作为一个有用的起点。

针对这本书，我强烈建议按顺序阅读，因为每一章都依赖前一章。对每一章，我推荐以下步骤。

&nbsp;
### 1) 第一次阅读（离线）

建议从头到尾先读完该章，此时不要写代码。第一次通读的目标是先把大局观理清。

理想情况下，建议离开电脑阅读。纸质书很好，但在无干扰的数字设备（无浏览器、社交媒体或邮件）上阅读也行。

我个人既读纸质书也用电子墨水平板。从 2018 年起我就用电子墨水设备，并总想多在上面阅读，不过我仍然发现纸质书能让我更专注。这也是为何我有时会打印那些难啃、或特别想读懂的研究论文。

我建议把第一次通读控制在短而专注的 20 分钟左右，尽量减少干扰，不要纠结或卡在细节上。

划出或注释让你困惑或感兴趣的部分没问题，但此阶段我不会去查资料。我只是建议先读，不运行任何代码。第一次通读是为了理解整体脉络。

&nbsp;
### 2) 第二次阅读（配合代码）

第二次通读时，我建议把章节中的代码敲出来并运行。直接复制很诱人，因为重打字很费劲，但当我读其他技术书时，亲手敲代码通常能让我多思考些（而不是只是扫一眼）。 

如果得到的结果与书中不同，我会去看看本书的 GitHub 仓库并尝试那里的代码。如果仍然不同，我会检查是否是包版本、随机种子、CPU/CUDA 等差异。如果还是弄不清，向作者提问也不是坏主意（通过书本论坛、公开 GitHub Issues 或 Discussions，实在不行再发邮件）。

&nbsp;
### 3) 练习

完成第二次通读、敲过代码并运行之后，通常是尝试练习题的好时机。它有助于巩固理解，或以半结构化的方式琢磨一个问题。如果练习太难，参考解答也可以。不过我仍建议先认真尝试一下。

&nbsp;
### 4) 回顾笔记并进一步探索

现在，在读完章节、跑过代码、做完练习后，建议回到前两次阅读中的高亮和批注，看看是否还有不清楚的地方。

这也是查阅额外参考资料或快速搜索以澄清未解之处的好时机。即便一切都明白，多读读自己感兴趣的主题也不错。

在这个阶段，把有用的想法、代码片段等写下或转存到你喜欢的笔记应用里也很有意义。 

&nbsp;
### 5) 在项目中使用这些想法

前面的步骤都是在吸收知识。现在，看看能否在自己的项目中使用某一章的部分内容。或者基于书中的代码构建一个小项目。想找灵感的话，看看额外资料，那些基本上是我为满足好奇心做的小项目。

例如，在读完多头注意力机制并实现了 LLM 后，你可能会想带有分组查询注意力的模型表现如何，或者 RMSNorm 相比 LayerNorm 真的差多少，等等。

也可能有更小的细节能用在自己的项目里。比如有时一个微小的细节就很有用，比如测试显式调用 `torch.mps.manual_seed(seed)` 是否与只用 `torch.manual_seed(seed)` 不同。

不过最终，我总想把这些知识用起来。这可能涉及使用该章的主要概念，有时也可能是沿途学到的小点子，比如即便是检查在项目里显式调用
`torch.mps.manual_seed(seed)` 真的是否比只用 `torch.manual_seed(seed)` 有区别这样的琐事。

&nbsp;
### 补充想法

当然，上述建议并非一成不变。如果主题整体很熟或很简单，而我主要是为了获取后面章节的信息，那么略读一章也没问题（以免浪费时间）。

另外，对于没有任何代码的章节（例如第 1 章导论），跳过与代码相关的步骤当然是合理的。

希望这些建议对你有帮助，祝阅读和学习愉快！
